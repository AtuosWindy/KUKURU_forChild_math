・ API・リクエストの基本的な流れについて、改めてお勉強の時間を設けた。(下記、お勉強の成果を記す。)
	⇒ ブラウザが URL「/xxx」を、サーバ(python)にリクエストする
	　 → XXX.py (/routers/.) の FastAPI (/xxx) が "/xxx.html" を返す。
	　 → ブラウザが "/xxx.html" を表示
	　 → .html の中で js (/static/app.js) を呼ぶ
	　 → js (/static/app.js) の中で XXX.py (/routers/.) の "/api/XXX" を呼ぶ
	　 → "/api/XXX" が ( データをセッションで保存・データ更新し！ ) json 形式の戻り値を返す
	　 → /xxx.html にリダイレクトし、/xxx.html を表示(または表示が変更される)

・ JS ⇔ Python のデータのやりとりの見直し・お勉強を行った↓
	: PythonからJSに値を渡す場合 - > /api/xxx にて return{} した値が、js側で const yyy = await fetch("/api/xxx") で取得できる。
																	→さらに、const data = await yyy.json() とすることで、data.xxx などで値を取得できる。
	  JSからPythonに値を渡す場合 -> js側で const data = { xxx: xxx, yyy: yyy, ... } として、fetch("/api/xxx", { method: "POST", body: JSON.stringify(data) }) とする。
																							Python側では、@app.post("/api/xxx") の関数の引数に data: dict として受け取ることができる。
