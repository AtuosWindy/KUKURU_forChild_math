※構想
[ページ]
/            →	トップページ。学年選択画面。ここで選択した学年はセッションに保存され、/home以降の画面で「学年別」ランキングを表示する際に利用する。
/home        → 学年・単元・難易度選択画面。(/home?grade=x&subject=y&difficult=z→/problemへ遷移)
/tutorial    → チュートリアル画面(各単元の説明,解き方の説明的な画面。要はプチ授業)
/start       → 問題生成
/problem     → 問題画面
/result      → 結果画面
/ranking	 → ランキング画面

/api/problem           問題データ取得
/api/answer            回答送信
/api/result            結果取得
/api/ranking           ランキング取得
/api/register          ランキング登録


※画面遷移の構想
[/]
説明：本アプリのトップページ。
　　　学年をプルダウンorスクロールバー(これから実装)で選択し(デフォルトは空欄、下に1~6の数字がある)、
　　　選択して「はじめる」ボタンを押すと、/home へ遷移する。

　　　この / で選択した学年はセッション(your_grade)に保存され、/home 以降の画面で「学年別」ランキングを表示する際に利用する。

　　　なお、ここでデフォルトの空欄のまま「はじめる」ボタンを押した場合も /home へ遷移し、通常通り利用できるようにする。
　　　その場合、ランキングは「総合」ランキングが表示されるようにする
　　　(デフォルトが/ranking?kind=allとなり、また「学年別」への切り替えタブが無効になる)。

　　　ただ、学年をここで選択しない場合でも、/result でランキング登録する際に、学年を入力できるようにする(任意入力)。
　　　その場合、ランキング登録の際に入力した学年が「学年別」ランキングに反映されるようになり、
　　　次に/rankingに移動した際はデフォルトが/ranking?kind=your_gradeに戻る。
　　　(セッションのyour_gradeの値も更新される設計を考えている)


[home]
説明：各画面にて、学年・単元・難易度を選択することで、/problem へ遷移する。

　　　難易度と同階層にある「おべんきょう」を選択した場合は、/tutorial へ遷移する。

　　　学年・単元・難易度の選択画面では、「もどる」「ホームへ」ボタンを実装する。

/home (学年>単元>難易度を選択)→ /problem
/home (学年>単元>"おべんきょう")→ /tutorial
/home (ランキング)→ /ranking
/home : 各選択画面に「もどる」・「ホームへ」ボタンを実装


[problem]
説明：/home で選択した内容に基づいて問題を生成し、出題する。最後まで問題を解き終わったら、/result へ遷移する。

　　　間違えた問題があった場合は、「間違えた問題を解きなおしますか？[はい]/[いいえ]」と聞き、
　　　"はい"を選択した場合は、間違えた問題を順に出す。再び間違えたとしても、/result へ遷移する。
　　　"いいえ"を選択した場合は、/result へ遷移する。

　　　途中でやめる場合や分からない問題をスキップしたい場合に備えて、
　　　/problem 画面に「もどる」「スキップ」ボタンを実装する。

　　　なお、/problem の「もどる」ボタンは、/home へ遷移する。「スキップ」ボタンは、次の問題へ進む。

/problem : difficulty==1 → かんたん問題を出す(４択の選択式)
/problem : difficulty==2または3 → 標準、または難しい問題を出す(入力式)
/problem (最後まで問題を解く、かつ全問正解) → /result
/problem (最後まで問題を解く、かつ間違いあり) → "間違えた問題を解きなおしますか？[はい]/[いいえ]"で "いいえ" → /result
/problem (最後まで問題を解く、かつ間違いあり) → "間違えた問題を解きなおしますか？[はい]/[いいえ]"で "はい" → 間違えた問題を順に出す → (再び間違えたとしても) /result
/problem (途中で"スキップを押す") → 次の問題へ(ページは変わらず)
/problem (途中で"もどるを押す") → 「ほんとうにやめますか？[はい]/[いいえ]」で "はい" → /homeへ
/problem (途中で"もどるを押す") → 「ほんとうにやめますか？[はい]/[いいえ]」で "いいえ" → 問題続行(表示が出ている間は回答ボタンや入力欄に触れられない)


[result]
説明：/problem で解いた問題の結果を表示する。正解数や正解率、平均解答時間を表示する。
　　　ユーザーは任意でランキングに登録できるようにする。ランキング登録画面では、
　　　ニックネーム入力欄・学年入力欄(空欄時のみ入力が有効。デフォルトは / で保存した your_grade の値が入力されており、空欄でない場合は入力不可となる)と、
　　　単元名、解いた問題数、正解率、平均解答時間を表示する。
	

/result : /problem で解いた問題の結果を表示する(正解数、正解率、平均解答時間)
/result ("ランキング登録する"ボタン)→ニックネームの入力、学年・単元名・出題数・正答率・タイム・スコアを表示する
/result (ニックネーム入力して"登録"ボタン)→ ランキングに登録する → 自動遷移で /home へ。
/result (それぞれの問題がリストアップされていて、それぞれの問題で「ふりかえり」ボタンがある) → 問題画面に遷移して、ユーザの選択結果と回答を表示する。遷移先ではもちろん「もどる(/result)」を実装。


[ranking]
説明：ランキングは「学年別」と「総合」の切り替えで表示できるようにする。
	　
	　ルートとしては、/ranking?kind=your_gradeまたは/ranking?kind=allを考えている。
	　デフォルトが「学年別」の表示を考えているので、/ranking?kind=your_gradeがデフォルトで表示されるようにする。
	　なお、いたずら防止でyour_gradeが表に出ないようにしたいので、/ranking?your_grade=1~6のような形はとりたくなく、
	　dbから取得した値を /ranking で表示としたい。

	　「総合」ランキングはTOP100のみ表示するなど制限をかける。

	　ランキング切り替えは、スイッチのようなUIで「学年別」⇔「総合」を切り替えられるようにする。
	　(デフォルトは「学年別」の方になる。切り替えた側の/ranking?kind=(選択したランキング)に移るようにする)
	　また、「ホームへ」ボタンは常時表示するようにする。

	　また、できれば！の話だが、、、本当は「学年別」ランキングは、
	　自分の周囲のランキングを表示するなど、子供たちが自分と同じくらいのレベルの人と競えるようにしたい。
	　(このアプリの設計上できるなら、、、だけど。)
	　もしそれが叶わないようなら、以下の実装方針で考えている。

	　下記、別案だが、
　　　デフォルトは「学年別」で表示され、プルダウンタブで
　　　top「～100」「～300」「～500」「～700」「～1000」「～5000」「～10000」を選択して表示、という別方針案もある。
	　(～10000はでかい気もするけど、、、まぁ興味本位でねｗ実装はしてみたくなるもんです。まぁ後から追加実装できるだろうし。)

/ranking(デフォルトは学年別表示) : 学年別ランキングを表示する(自分の学年のランキングを表示する)
/ranking(切り替えタブの操作で総合表示可能) : 総合ランキングを表示する(TOP100のみ表示する)
/ranking(学年別にて) (ランキング切り替えで"総合ランキング"を選択) → 総合ランキングを表示する
/ranking(総合にて) (ランキング切り替えで"学年別ランキング"を選択) → 学年別ランキングを表示する
/ranking	("ホームへ"ボタン)　→ /home へ遷移
(/rankingにて学年の入力欄を設け、入力したら学年別を解禁しようかとも考えたが、、学年の入力がない＆問題を解ききってないユーザということになるので、閲覧制限になっちゃうけどまぁいっかなって思ってる。自分が努力せずして他人の華を見るようにはなってほしくない。)
